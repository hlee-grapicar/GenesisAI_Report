# 개념 숙지

### environment
- 시뮬레이션 및 테스트가 실행되는 환경
- 상태(관측결과) 제공, 행동 수용, 보상 제공, 상태 전이

### step
- 에이전트와 환경 간의 상호작용이 한 번 진행되는 최소 단위
- 에이전트의 관측 -> 에이전트의 행동 결정 -> 행동 실행 및 환경 전이 -> 보상 및 다음 상태 전달 의 과정

### rollout
- 현재 정책이 환경과 상호작용하며 경험 데이터를 생성하는 과정
- 예제에서는 num_steps_per_env: 24로 설정되어, 24 step이 완료되면 하나의 rollout이 생성됨(4096의 병렬환경이 설정되어있어 실제로는 24 * 4096개 생성)
- 생성된 rollout은 학습 단계에서 사용

### transition
- 하나의 step이 발생하고 나서 수집되는 경험 데이터의 묶음
- transition이 모여 rollout을 구성함

### iteration
- 데이터 수집과 정책 업데이트가 한 번 완료되는 주기
- 롤아웃 배치 생성(수집) -> 정책 및 가치 네트워크 학습(학습) 으로 이루어짐
- 현재 코드는 max_iterations=101 로 설정되어, 총 101번의 iteration이 반복되면서 학습을 해 나감

### mini-batch processing
- 대량의 학습 데이터를 작고 관리하기 쉬운 묶음으로 나누어 순차적으로 처리하고 학습하는 기법
- 메모리 효율성, 안정적 학습, 병렬 처리 가능
- 현재 코드는 24*4096 transition으로 구성된 rollout을 num_mini_batches: 4 에 따라 4개의 mini-batch로 나누어 신경망 업데이트

### epoch
- 전체 학습 데이터셋을 한 번 완전히 통과하여 학습하는 과정
- 현재 코드에서 4개의 mini batch가 각각 가중치 업데이트 한 번씩 이루어지면 1 에포크가 완료
- 현재 코드에서 num_learning_epochs: 5 설정에 의해 같은 rollout 데이터로 총 5번을 학습
